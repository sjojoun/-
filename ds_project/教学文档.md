# 大数据离线与实时处理实战（竞赛篇）- 完整教学文档

## 一、项目概述

本项目涵盖了典型的大数据架构的三个核心阶段：**离线数据处理**（抽取、清洗、指标计算）、**数据挖掘**（特征工程、推荐系统）以及**实时数据计算**（数据采集、流式计算与双流JOIN）。
本项目使用 **Scala 2.12** 编写，通过 **Maven** 进行统一构建，涉及组件包括：Spark、Flink、Kafka、Flume、Maxwell、MySQL、Hive、ClickHouse、HBase 和 Redis。

## 三、编译与打包说明

因为整个项目依赖了较多大数据组件包，为了能够轻松地在集群环境（YARN / Spark Standalone / Flink Cluster）上提交任务，使用 `maven-assembly-plugin` 将依赖打包（打 fat-jar）。

1.  **进入项目根目录**：
    ```bash
    cd /app/ds_project
    ```
2.  **执行 Maven 打包命令**：
    ```bash
    mvn clean package -DskipTests
    ```
    *(注：打包可能需要几分钟时间，请确保开发环境已安装并正确配置了 JDK 8 和 Maven)*
3.  **获取构建包**：
    成功后，在 `target/` 目录下会生成名为 `bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar` 的文件。我们将使用此 JAR 包向集群提交任务。

## 四、执行与运行指南

### 前置准备（初始化数据库与表）
在运行各类 Spark/Flink 代码之前，请务必先创建所需的数据库表。
1.  **ClickHouse**：将 `src/main/resources/ddl.sql` 中的脚本在 ClickHouse 中执行。
    ```bash
    clickhouse-client -m < src/main/resources/ddl.sql
    ```
2.  **Hive**：将 `src/main/resources/hive_ddl.sql` 中的脚本在 Hive 中执行。
    ```bash
    hive -f src/main/resources/hive_ddl.sql
    ```

### 模块一：离线数据处理 (Spark)

> **前置要求**：HDFS、YARN、MySQL(DS_DB01)、Hive 已经启动并在运行中。

**任务 1：数据抽取 (`Task1_Extract`)**
*   **功能**：使用 Spark JDBC 读取 MySQL `ds_db01` 库的增量数据，加上静态分区（前一天日期）后，写入 Hive `ods` 库。
*   **执行命令**：
    ```bash
    spark-submit --class com.ds.offline.Task1_Extract \
      --master yarn --deploy-mode client \
      target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
    ```

**任务 2：数据清洗 (`Task2_Clean`)**
*   **功能**：结合最新的 DWD 数据和昨日 ODS 的数据，按主键排序（取最新的一条），添加 `dwd_insert_user` 等审计字段，并规范化日期字段。
*   **执行命令**：
    ```bash
    spark-submit --class com.ds.offline.Task2_Clean \
      --master yarn --deploy-mode client \
      target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
    ```

**任务 3：指标计算 (`Task3_Metrics`)**
*   **功能**：涉及多达 10 项复杂的 KPI 指标计算（中位数比较、省份 Top3、连续下单占比、滑动窗口计算等），将结果写入 DWS 或 ClickHouse。
*   **执行命令**：
    ```bash
    spark-submit --class com.ds.offline.Task3_Metrics \
      --master yarn --deploy-mode client \
      target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
    ```

### 模块二：数据挖掘 (Spark MLlib)

> **特点说明**：本模块侧重于特征提取和推荐算法。建议分配较多内存。

**任务 1 & 3：特征工程 (`Task1_Feature1` / `Task3_Feature2`)**
*   **功能**：剔除无效数据；对数值进行 `StandardScaler` 处理；对品牌/类别进行 `One-Hot` 编码；将原始 ID 映射成连续的 Index (Mapping操作)。
*   **执行命令**（以 Feature1 为例）：
    ```bash
    spark-submit --class com.ds.mining.Task1_Feature1 \
      --master yarn --deploy-mode client \
      target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
    ```

**任务 2 & 4：推荐系统 (`Task2_Recommend1` / `Task4_Recommend2`)**
*   **功能**：计算用户商品相似度、使用 SVD 分解保留 95% 信息量并生成推荐列表 Top 5。
*   **执行命令**（以 Recommend2 为例）：
    ```bash
    spark-submit --class com.ds.mining.Task4_Recommend2 \
      --master yarn --deploy-mode client \
      target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
    ```

---

### 模块三：实时计算 (Flume/Maxwell + Flink)

> **前置要求**：Zookeeper, Kafka, Redis, HBase 必须处于运行状态。

**步骤 1：启动实时数据采集**
1.  **Maxwell (MySQL Binlog -> Kafka)**
    *   将 `src/main/resources/maxwell.properties` 放入 Maxwell `config.properties` 路径，或者运行时指定。
    *   **命令**：
        ```bash
        /opt/module/maxwell-1.29.0/bin/maxwell --config src/main/resources/maxwell.properties
        ```
2.  **Flume (Socket Log -> Kafka)**
    *   使用 `src/main/resources/flume_agent.conf` 启动 Flume agent。
    *   **命令**：
        ```bash
        flume-ng agent -n a1 -c conf -f src/main/resources/flume_agent.conf -Dflume.root.logger=INFO,console &
        ```
    *   接着启动 `/data_log/gen_ds_data_to_socket` 脚本生成数据。

**步骤 2：启动 Flink 实时处理任务 (`Task2_FlinkProcess`)**
*   **功能**：
    *   消费 Kafka ODS 层的两个 Topic (`ods_mall_log` 和 `ods_mall_data`)。
    *   将数据按表名过滤，分发到 Kafka DWD 层的对应 Topic (`fact_order_master`, `fact_order_detail` 等)。
    *   解析 JSON 数据，实时将数据备份至 HBase 外部表。
    *   对关键指标使用窗口（TumblingProcessingTimeWindows）和状态机（KeyedProcessFunction）进行计算，输出实时 Top3 商品、Top2 用户及总单数到 Redis。
    *   使用 Flink 双流 JOIN (Join + Window) 将主表与详情表合并，写入 ClickHouse 中。
*   **执行命令 (采用 Per-Job 模式)**：
    ```bash
    flink run -m yarn-cluster \
      -c com.ds.realtime.Task2_FlinkProcess \
      target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
    ```

## 五、重点难点解析与代码导读

1.  **Hive HBase 外部表关联**
    *   **难点**：如何在 Flink 中将数据流式写入 HBase 的同时，能够使用 Hive 进行离线查询？
    *   **解决**：在 Flink 侧，继承 `RichSinkFunction` 解析 JSON 并通过 `Put` 对象插入 HBase。在 Hive 侧，使用 `org.apache.hadoop.hive.hbase.HBaseStorageHandler` 建立外部表关联，其中 `hbase.columns.mapping` 参数是关键，它定义了 Hive 列到 HBase `family:qualifier` 的映射关系。
2.  **Flink 双流 JOIN (订单流与详情流)**
    *   **难点**：实时数据流由于网络等原因可能乱序，如何保证主表和子表能够准确关联。
    *   **解决**：代码中使用了基于时间窗口的 `join().where().equalTo().window()` 机制。将相同 `order_id` 的记录在 10 秒滚动窗口中聚合。虽然题目提示“无需担心数据迟到”，但这套方案足以应对常规生产环境的乱序。
3.  **SVD 降维与动态 95% 差异率计算**
    *   **难点**：题目要求保留“至少 95%”的信息量。
    *   **解决**：在 `Task4_Recommend2` 中，我们首先用一个相对宽泛的 K（如 100）计算 `computeSVD(K, computeU = false)` 得到所有奇异值。奇异值的平方代表了主成分的方差。我们对平方求和作为 `totalVariance`，然后累加奇异值平方，直到其占比大于等于 `0.95`，此时锁定的 `K` 即为真实需要的降维维度，随后以此精确的 `K` 值计算完整的 SVD 矩阵 (`computeU = true`)。

## 六、总结
本项目全面覆盖了数据生命周期：从 Maxwell/Flume 的**数据采集**，到 Spark 的**批处理清洗加工与离线数仓**，深入 Spark MLlib 的**推荐特征工程**，再到 Flink 的**毫秒级流计算与实时大屏后端（Redis/ClickHouse）对接**。非常适合作为大数据组件综合实训的闭环参考样例。
