# 大数据实战竞赛 - 模块一：项目结构与环境说明

## 1. 软件环境与技术栈
本项目基于标准的大数据竞赛环境构建，主要包含：
*   **操作系统**: CentOS 7 / Ubuntu
*   **开发语言**: Scala 2.12.10, Java 8
*   **底层集群组件**: Hadoop 3.1.3, ZooKeeper 3.5.7, YARN 3.1.3
*   **计算框架**:
    *   **Spark 3.1.1** (用于离线数据处理、数据清洗、指标计算、机器学习推荐算法)
    *   **Flink 1.14.0** (用于实时数据流接入、窗口统计、Redis Sink、双流 Join)
*   **存储与中间件**:
    *   **MySQL 5.7**: 存放业务源数据 `ds_db01` 及商品维表 `sku_info`。
    *   **Hive 3.1.2**: 离线数仓存储 (`ods`, `dwd`, `dws`)，外挂 HBase 表。
    *   **HBase 2.2.3**: 实时数据备份与多维查询引擎。
    *   **ClickHouse 21.9.4**: 联机分析处理 (OLAP) 的终端数据存储，存放 `shtd_result` 结果表。
    *   **Redis 6.2.6**: 高速键值缓存，用于 Flink 实时统计的 Top-N 排行榜输出。
    *   **Kafka 2.4.1**: 消息中间件，存放由 Flume 和 Maxwell 采集的实时数据流 (`ods_mall_log`, `ods_mall_data`)。

## 2. 源码文件架构清单与说明

整个项目采用 Maven 进行统一管理，按照业务模块切分为 `offline`（离线）、`mining`（挖掘）和 `realtime`（实时）。
以下是项目中每一个文件的具体用途和职责：

```text
bigdata-competition/
├── pom.xml
│   # Maven 构建脚本。定义了 Scala, Spark, Flink 及所有的第三方 Connector (Kafka, JDBC, HBase, Redis, ClickHouse, FastJSON) 依赖版本，同时配置了 assembly 插件以支持打 fat-jar，便于通过 spark-submit 或 flink run 直接提交到集群。
│
├── docs/
│   # 教学说明文档存放目录（即本文档所在位置）。
│   ├── 01_项目结构与环境说明.md
│   ├── 02_离线数据处理教学.md
│   ├── 03_数据挖掘教学.md
│   └── 04_实时数据处理教学.md
│
├── src/main/resources/
│   # 资源与配置文件目录。
│   ├── ddl.sql                    # 存放 ClickHouse `shtd_result` 库下所有宽表的建表语句 (MergeTree 引擎)。
│   ├── hive_ddl.sql               # 存放 Hive `ods` 等数仓库的创建语句，以及与 HBase 绑定的外部表创建语句。
│   ├── flume_agent.conf           # Flume 的 Agent 配置。监听 netcat (25001) 将 socket 日志数据实时投递给 Kafka `ods_mall_log`。
│   ├── maxwell.properties         # Maxwell 的配置文件。监控 MySQL binlog，筛选出 `ds_db01` 库的插入更新操作，发送到 Kafka `ods_mall_data`。
│   └── log4j.properties           # 控制台日志格式配置文件，降低运行时 INFO 级别日志刷屏。
│
└── src/main/scala/com/ds/
    # Scala 核心源代码目录。
    ├── offline/                   # 【模块：离线数据处理】
    │   ├── Task1_Extract.scala    # 从 MySQL 增量读取前一天的数据并作为动态/静态分区导入到 Hive `ods` 层。
    │   ├── Task2_Clean.scala      # 将 `ods` 的数据和数仓 `dwd` 最新分区的数据按主键进行聚合/合并处理，规范化时间格式，补充四大审计字段(insert_user等)，形成 `dwd` 明细层宽表。
    │   └── Task3_Metrics.scala    # 基于 `dwd` 层事实表与维度表，通过 Spark SQL / DataFrame API 进行 10 大复杂 KPI 指标的聚合计算，并将最终排名写入 ClickHouse 及 Hive `dws` 表。
    │
    ├── mining/                    # 【模块：数据挖掘】
    │   ├── Task1_Feature1.scala   # 【特征工程 1】提取 MySQL SKU 数据，进行 StandardScaler 规范化与类别特征的 One-Hot 编码转换。
    │   ├── Task2_Recommend1.scala # 【推荐系统 1】根据共同购买商品种类找到 Top10 相似用户，计算候选商品与目标用户的余弦相似度并推荐 Top5。
    │   ├── Task3_Feature2.scala   # 【特征工程 2】对 customer_id 和 product_id 进行连续整数映射（Mapping），并对用户购买记录行转列，构建用户-商品 One-Hot 特征矩阵。
    │   └── Task4_Recommend2.scala # 【推荐系统 2】基于 Spark MLlib 的 SVD 矩阵分解算法，动态截取累计贡献率大于 95% 的主成分（奇异值），对用户进行降维空间的相似商品推荐。
    │
    └── realtime/                  # 【模块：实时数据采集与计算】
        └── Task2_FlinkProcess.scala # Flink DataStream API 入口类。实现消费 Kafka、按表名分发 DWD Topic、记录备份进 HBase、计算实时订单总量与排行(存入 Redis)、监控退款存入 ClickHouse 以及基于 10秒 滚动窗口的双流订单主子表 JOIN 操作。
```

## 3. 项目编译指令
在集群的主节点上，进入工程根目录，执行：
```bash
mvn clean package -DskipTests
```
成功后，所有的业务逻辑及依赖组件都会被打包至 `target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar` 中。之后各章节的具体任务，只需使用 `spark-submit` 或 `flink run` 指定这个 jar 包中对应的 Main Class 即可运行。
