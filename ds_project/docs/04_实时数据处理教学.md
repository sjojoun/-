# 大数据实战竞赛 - 模块四：实时数据处理教学

本模块基于 **Flink 1.14** DataStream API 开发。负责处理由 Kafka 接收的实时电商事件数据，包括用户日志与订单明细。

## 1. 实时采集 (Flume & Maxwell)
### 1.1 Maxwell 配置
Maxwell 读取 MySQL 的 Binlog 转化为 JSON。我们在 `src/main/resources/maxwell.properties` 配置了:
```properties
producer=kafka
kafka.bootstrap.servers=master:9092
kafka_topic=ods_mall_data
filter=include:ds_db01.*
```
启动 Maxwell：`/opt/module/maxwell/bin/maxwell --config conf/maxwell.properties`

### 1.2 Flume Socket 日志采集
使用 `src/main/resources/flume_agent.conf` 启动一个 netcat source 接收本机 25001 端口的数据，输出到 Kafka (`ods_mall_log`):
```properties
a1.sources.r1.type = netcat
a1.sources.r1.bind = master
a1.sources.r1.port = 25001
```
启动 Flume：`flume-ng agent -n a1 -c conf -f conf/flume_agent.conf &`

## 2. 任务二：实时流处理 (`Task2_FlinkProcess.scala`)
### 需求分析
这是一个功能极为丰富的 Flink 流式程序，涉及：
1. **分发过滤 (Kafka Source/Sink)**：消费 ODS 层的 JSON 数据，通过分析 `table` 字段，提取 `data` 分发到事实表 `fact_order_master`, `fact_order_detail`，以及用户登录日志 `dim_customer_login_log` 的 DWD 层对应 Kafka 主题。
2. **实时 HBase 备份**：将实时数据解析成 HashMap，再转换为宽表数据，写入外部 HBase 引擎，并通过 Hive External Table 完成联机查询 (`ods` 库)。
3. **订单金额/销量 TopN 排行 (Redis Sink)**：统计订单量 (`totalcount`)、商品销售量 Top 3 (`top3itemamount`)、商品销售额 Top 3 (`top3itemconsumption`) 和消费前 2 的用户 (`top2userconsumption`)。
4. **异常订单过滤 (ClickHouse Sink)**：过滤出 `order_status = 2` 的已退款订单，直接利用 JDBC `PreparedStatement` 保存至 ClickHouse 分析集群。
5. **双流基于窗口的 JOIN**：将订单主表 (OrderMaster) 与订单详情 (OrderDetail) 通过相同的 `order_id` 进行关联拼接，输出按时间窗口聚合的结果，最后存入 ClickHouse。

### 核心代码讲解
*   **JSON 数据解析分发**：使用 `com.alibaba.fastjson.JSON` 转换并过滤出目标表结构。利用 Flink 的 `map` 和 `addSink(new FlinkKafkaProducer)` 重新发送给 Kafka DWD Topic。
*   **HBase 外部接入点**：自定义继承 `RichSinkFunction` 的 `HBaseSink`，在 `open` 生命周期里创建 `HBaseConfiguration.create()` 与 `ConnectionFactory`。并重写 `invoke` 采用 `Put` 对象向行键(`rowKeyStr` 为随机数加日期) 添加所有 JSON 提取的列 (`put.addColumn`)。
*   **Top-N 状态机**：
    对于实时排行榜，简单的 `keyBy` 是不够的。我们定义了带有一个共享全局键 (`keyBy(_ => "all")`) 的 `KeyedProcessFunction`：
    *   在内部维护一个 `scala.collection.mutable.HashMap` 用来记录每个 商品/用户 的累计值。
    *   每次有新数据到达时，更新字典中的统计量。
    *   将字典 `.toSeq.sortBy(-_._2).take(N)` 取出前 N 的键值对（例如：`"1:10020.2,42:4540.0"`），封装成题要求的 `[ ]` 格式，然后发送至 Redis 的 `Jedis.set()` 进行覆盖。
*   **Redis Sink**：重写 `RichSinkFunction[String]` 的 `open` 中初始化 `Jedis("master", 6379)`。
*   **双流基于 Window 的 Join**：
    为了将拥有相同订单 ID 的明细拼在一起（例如同一个单买了两件商品）：
    ```scala
    omStream.join(odStream)
      .where(_.orderId)
      .equalTo(_.orderId)
      .window(TumblingProcessingTimeWindows.of(Time.seconds(10)))
      .apply { (om, od) => (om.orderSn, om.orderMoney, od.productCnt) }
    ```
    此段代码不仅处理了两个异步流的关联（默认 10秒内的乱序），还顺带执行了对关联后结果的归纳处理，随后进入下一个 `reduce` 阶段按 `sn` 把所有商品数量 (`productCnt`) 累积在一起，然后发送到 ClickHouse 归档。

### 执行命令
*(注：建议将 Flink 集群在 YARN 上启动一个 Session 或直接以 Per-Job 模式提交。)*
```bash
flink run -m yarn-cluster \
  -c com.ds.realtime.Task2_FlinkProcess \
  target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
```
