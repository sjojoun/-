# 大数据实战竞赛 - 模块四：实时数据处理教学

## 一、 模块概述

本模块的核心目标是利用 **Flink (1.14.0)** 流式计算引擎，配合 **Flume** 和 **Maxwell** 等数据采集组件，构建一个高吞吐、低延迟的实时数据处理管道。

系统整体流程如下：
1. **数据源采集**：
   * **日志数据**：利用 Flume 从特定的 socket 端口（25001）监听并抓取实时生成器（`gen_ds_data_to_socket`）的日志，发送至 Kafka 的 `ods_mall_log` 主题。
   * **业务数据**：利用 Maxwell 监听 MySQL 的 Binlog（二进制日志），捕获 `ds_db01` 库的数据变更（如新增的订单、订单详情），发送至 Kafka 的 `ods_mall_data` 主题。
2. **Flink 实时分发与持久化**：
   * Flink 消费 Kafka 中的原始 ODS 级数据。
   * **数据分流**：根据 JSON 中的表名标志，将数据拆分投递至对应的 Kafka DWD 层主题（如 `fact_order_master`, `dim_customer_login_log`）。
   * **备份归档**：将解析出来的原始数据以列族的形式实时 `Put` 进 HBase 的外部表中，供 Hive 建立外部表做离线联机查询。
3. **Flink 实时聚合指标计算**：
   * **Redis 榜单**：使用带状态计算（KeyedProcessFunction）计算总订单量、商品销量 Top3、商品销售额 Top3、用户消费 Top2 等大屏指标，格式化输出后覆盖写入 Redis。
   * **ClickHouse 异常与大宽表汇聚**：过滤并捕获已退款的订单信息写入 ClickHouse；此外，将订单主表流（OrderMaster）与订单明细流（OrderDetail）利用 **10 秒滚动时间窗口** (`TumblingProcessingTimeWindows`) 进行双流 JOIN，聚合完成后直接打入 ClickHouse 的聚合宽表 (`shtd_result.orderpostiveaggr`) 中。

## 二、 环境准备与配置教程

在启动 Flink 任务之前，必须确保底层数据通道和采集工具正常工作。

### 1. 启动 ZooKeeper 和 Kafka
由于 Flink 依赖 Kafka，而 Kafka 依赖 ZooKeeper，所以必须先启动这两个服务：
```bash
# 启动 Zookeeper（假设您的路径在此）
zkServer.sh start

# 启动 Kafka（指定后台运行）
kafka-server-start.sh -daemon /opt/module/kafka-2.4.1/config/server.properties
```

**创建必要的 Kafka Topic**：
虽然较新版本的 Kafka 支持自动创建，但在正规竞赛中，通常要求手工注册，并指定分区数为 4。
```bash
# 贴源层 (ODS) Topic
kafka-topics.sh --create --bootstrap-server master:9092 --replication-factor 1 --partitions 4 --topic ods_mall_log
kafka-topics.sh --create --bootstrap-server master:9092 --replication-factor 1 --partitions 4 --topic ods_mall_data

# 事实与明细层 (DWD) Topic
kafka-topics.sh --create --bootstrap-server master:9092 --replication-factor 1 --partitions 4 --topic fact_order_master
kafka-topics.sh --create --bootstrap-server master:9092 --replication-factor 1 --partitions 4 --topic fact_order_detail
kafka-topics.sh --create --bootstrap-server master:9092 --replication-factor 1 --partitions 4 --topic dim_customer_login_log
```

### 2. 启动 HBase 与 ClickHouse
HBase 用于实时备份，ClickHouse 用于存放过滤异常和 JOIN 的结果集。
```bash
# 启动 HBase
start-hbase.sh

# 启动 ClickHouse 服务
systemctl start clickhouse-server
```

### 3. 配置并启动 Flume 采集日志数据
Flume 的职责是将日志生成器输出到本机 25001 端口的数据抓取到 Kafka 中。
我们在项目的 `src/main/resources/flume_agent.conf` 中定义了一个 `netcat` Source 和一个 `KafkaSink`。

**执行命令启动 Flume Agent**：
```bash
# 请将终端切换到您的 Flume 目录，例如：
cd /opt/module/flume-1.9.0/

# 后台启动 flume
bin/flume-ng agent -n a1 -c conf/ \
  -f /app/ds_project/src/main/resources/flume_agent.conf \
  -Dflume.root.logger=INFO,console > /dev/null 2>&1 &
```

*(注意：Flume 启动后，您需要执行脚本启动数据生成器：`./gen_ds_data_to_socket`，如果在 `/data_log` 目录下，直接运行即可。)*

### 4. 配置并启动 Maxwell 采集 MySQL 业务数据
Maxwell 用于实时捕获 MySQL (ds_db01库) 产生的数据变更。我们在 `src/main/resources/maxwell.properties` 中指定了 `kafka_topic=ods_mall_data`，并且过滤只保留 `ds_db01.*`。

**执行命令启动 Maxwell**：
```bash
# 请将终端切换到您的 Maxwell 目录
cd /opt/module/maxwell-1.29.0/

# 启动 maxwell（使用项目提供的配置）
bin/maxwell --config /app/ds_project/src/main/resources/maxwell.properties > /dev/null 2>&1 &
```

## 三、 Flink 核心代码讲解与提交执行

经过前面的配置，数据已经源源不断地流入了 Kafka 的 `ods_mall_data` 和 `ods_mall_log`。接下来就是 Flink 大显身手的时候了。所有的代码均位于 `src/main/scala/com/ds/realtime/Task2_FlinkProcess.scala` 中。

### 代码难点剖析

**1. Kafka JSON 流的解析与分流**
```scala
val dwdOrderMasterStream = odsMallDataStream.filter { jsonStr =>
  JSON.parseObject(jsonStr).getString("table") == "order_master"
}.map { jsonStr => JSON.parseObject(jsonStr).getString("data") }
```
我们使用了 `com.alibaba.fastjson`。首先在 `filter` 中截取源自 `order_master` 表的数据包，然后在 `map` 中把 JSON 中包裹业务详情的 `"data"` 节点摘取出来，随后通过 `addSink()` 再输出到目标 Kafka 主题。

**2. 实时 Top-N 排行榜写入 Redis**
Flink 中普通的 `keyBy().sum()` 只能对单点做聚合。要做 Top-N 榜单，我们需要在所有的元素之上维护一个全局的数据字典。代码中我们巧妙地使用了：
```scala
productCountStream.keyBy(_ => "all").process(
  new KeyedProcessFunction[String, ProductCount, String] {
    val mapState = new scala.collection.mutable.HashMap[Int, Int]()
    override def processElement(value, ctx, out) = {
      // 1. 更新最新销售量
      mapState.put(value.productId, value.count)
      // 2. 将所有商品按销量降序排列，取前 3
      val top3 = mapState.toSeq.sortBy(-_._2).take(3)
                         .map(x => s"${x._1}:${x._2}").mkString(",")
      // 3. 按照题目要求封装成 [商品1:销量,商品2:销量] 的字符串下发
      out.collect(s"[$top3]")
    }
  }
).addSink(...) // 存入 Redis
```

**3. Flink 双流滚动窗口 JOIN**
如何把在不同时间产生的“订单主表（含总金额）”和“订单详细表（含所购商品数）”拼成一条记录？
```scala
omStream.join(odStream)
  .where(_.orderId)    // order_master 的主键
  .equalTo(_.orderId)  // order_detail 的外键
  // 开启 10秒 滚动窗口，把 10秒 内落入的同一订单的两条记录进行撮合
  .window(TumblingProcessingTimeWindows.of(Time.seconds(10)))
  .apply { (om, od) => (om.orderSn, om.orderMoney, od.productCnt) }
```
经过 `apply` 的撮合合并后，利用 `keyBy(_._1).reduce(...)` 根据订单序列号把所有挂载在这一单名下的子商品数量累加汇总，最终通过 `RichSinkFunction` 执行 JDBC 的 `PreparedStatement` 插入到 ClickHouse 中。

### 编译打包与任务提交

在完成代码编写后，我们要把 Flink 应用程序提交到集群（YARN 或 Standalone）执行。

**第一步：编译打包 (打 Fat-Jar)**
在项目根目录下，确保 Maven `pom.xml` 中包含了 Flink 相关的连接器（`flink-connector-kafka`, `flink-connector-jdbc` 等），然后执行：
```bash
cd /app/ds_project
mvn clean package -DskipTests
```
成功后，您将看到 `target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar` 文件。

**第二步：提交 Flink 任务**
使用 Flink 自带的命令行工具，向 YARN 提交一个 Per-Job 的应用，这样集群会在任务结束后自动回收资源。
```bash
flink run -m yarn-cluster \
  -c com.ds.realtime.Task2_FlinkProcess \
  target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
```
*（如果您的环境中 Flink 已经作为 Standalone 模式启动，可以直接 `flink run -c ...` 而不带 `-m yarn-cluster`。）*

本模块实现了“日志+数据库业务更新 -> 消息总线 -> 流处理过滤、双流 JOIN 和 TopN 聚合 -> Redis、ClickHouse 及 HBase 多元存储”，是典型的工业级 Lambda/Kappa 流处理架构落地实践。
