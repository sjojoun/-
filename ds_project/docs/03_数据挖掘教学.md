# 大数据实战竞赛 - 模块三：数据挖掘教学

本模块基于 **Spark MLlib** (Machine Learning Library) 构建。核心目标为利用电商平台的用户历史购买记录进行**特征工程**与**相似商品推荐系统**开发。所有代码位于 `com.ds.mining` 包下。

## 1. 任务一：特征工程 (一) (`Task1_Feature1.scala`)
### 需求分析
基于 `dwd` 的 `fact_order_detail`、`dim_customer_inf`、`dim_product_info` 表剔除孤儿数据（用户/商品不在维度表）。随后进行两大操作：
1. 计算与特定用户 (`customer_id` = 5811) 购买相同商品种类最多的前 10 位相似用户。
2. 对来自 MySQL 的商品记录表 (`sku_info`) 中的 6 个数值特征 (`price, weight, height, length, width, three_category_id`) 进行均值/方差的标准规范化 (StandardScaler)；并对分类特征 (`brand_id`) 进行 One-Hot 编码。将第一条记录的前 10 列按格式输出。

### 核心代码讲解
*   **寻找相似用户**：利用 DataFrame API 的 `join()` 求目标用户和其余所有用户购买商品集合的交集。用 `groupBy("customer_id").agg(count("product_id").as("same_product_count")).orderBy(desc)` 取前十。
*   **特征提取 Pipeline**：
    *   **VectorAssembler**: 将 6 个数值特征合并为一个 `Vector` 结构 (`numeric_features`)。
    *   **StandardScaler**: 配置 `setWithStd(true)` 和 `setWithMean(true)` 将该向量均值归零，方差置 1 (`scaled_features`)。
    *   **StringIndexer & OneHotEncoder**: 将分类变量 `brand_id` 转换为数值索引后，再打散成全 0/1 的稀疏向量 (`brand_vec`)，设置 `setDropLast(false)` 保留所有类别的列。
    *   **合并输出**：从 DataFrame 提取第一行，用 Scala Array 操作合并原始 `id`、归一化后数值特征以及独热向量并截取前 10 位，以逗号分隔打印。

### 执行命令
```bash
spark-submit --class com.ds.mining.Task1_Feature1 \
  --master yarn --deploy-mode client \
  target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
```

## 2. 任务二：推荐系统 (一) (`Task2_Recommend1.scala`)
### 需求分析
接任务一，从找到的前 10 位相似用户的购买记录中，**剔除目标用户 (5811) 已经买过的商品**，将剩下的商品作为候选推荐商品。接着，计算目标商品集和候选商品集在所有用户维度上的余弦相似度（Cosine Similarity），累加求均值，推荐均值最高的前 5 个商品。

### 核心代码讲解
*   **构建商品的用户向量表**：首先利用 `StringIndexer` 把所有用户 ID 转为 0 到 N 的索引，将每种商品被购买过的用户索引集合收集起来（`collect_list`），并存入 `SparseVector` 中（即构建该商品的用户特征向量）。
*   **余弦相似度计算**：
    *   使用 `org.apache.spark.ml.linalg.BLAS.dot(v1, v2)` 计算内积。
    *   使用 `Vectors.norm(v1, 2)` 计算欧几里得范数（二范数）。
    *   公式 `DotProduct / (Norm1 * Norm2)` 得到两个商品间的相似度。
*   **聚合与排序**：对每个候选商品，遍历目标商品集求平均相似度，最后通过 `sortBy(-_._2).take(5)` 获取推荐。

### 执行命令
```bash
spark-submit --class com.ds.mining.Task2_Recommend1 \
  --master yarn --deploy-mode client \
  target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
```

## 3. 任务三：特征工程 (二) (`Task3_Feature2.scala`)
### 需求分析
对 `fact_order_detail` 中的购买记录去重。首先，为用户 ID 和 商品 ID 分别生成一个从 0 开始自增的映射字典 (Mapping)。随后利用这个字典，将数据转化为 `customer_id` 为主键，其余所有 `product_idN` 作为列名的巨大用户商品特征矩阵，对于购买过该商品的值置 1，未购买的置 0。

### 核心代码讲解
*   **ID Mapping**：利用 Spark SQL 窗口函数 `row_number().over(Window.orderBy("customer_id")) - 1` 为不同主键生成连续整数，并将旧 ID 通过 `join` 映射到新 ID。
*   **行转列特征矩阵化 (One-Hot 行变换)**：利用 Spark SQL 提供的强大 `pivot()` 函数，将 `product_id` 行中的不同取值透视为列，并在 `.agg(first("value"))` 聚合时填入 1.0，后续通过 `.na.fill(0.0)` 把那些缺失的值补 0。为每一列重命名加上 `product_id` 前缀。

### 执行命令
```bash
spark-submit --class com.ds.mining.Task3_Feature2 \
  --master yarn --deploy-mode client \
  target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
```

## 4. 任务四：推荐系统 (二) (`Task4_Recommend2.scala`)
### 需求分析
利用 Spark RDD API (`org.apache.spark.mllib.linalg.distributed.CoordinateMatrix`) 对任务三提取的“用户-商品”评分矩阵（全为 1，隐式反馈）进行**奇异值分解 (SVD)**。保留能够代表原始数据 95% 信息的奇异值分量，获取降维后的商品矩阵，从而进行相似商品的余弦推荐。

### 核心代码讲解
*   **矩阵封装**：将 DataFrame 映射后的 `(cust_idx, prod_idx)` 转化为 `MatrixEntry(u, p, 1.0)` 并交给 `CoordinateMatrix`，进一步调用 `toRowMatrix()` 形成分布式的行矩阵。
*   **寻找 95% 信息量（截取前 K 维主成分）**：
    *   执行一次初步的 SVD `rowMat.computeSVD(maxK, computeU = false)` 得到特征值向量 `s`。
    *   信息量（方差）等比于奇异值的平方。所以我们将所有 `s` 的平方求和作为总方差 `totalVariance`。
    *   遍历截取奇异值，直到部分奇异值的平方累加和 `>= 0.95 * totalVariance`，记录此时的 `K`。
*   **提取特征空间商品向量**：使用计算出的精准 `K` 重新执行 `computeSVD` 并且 `computeU = true`。将返回的 V 矩阵 (`d x k`) 转置拆解成一个个对应各个商品 (`d`) 的 K 维特征向量，再依此向量做精确相似度计算。

### 执行命令
```bash
spark-submit --class com.ds.mining.Task4_Recommend2 \
  --master yarn --deploy-mode client \
  target/bigdata-competition-1.0-SNAPSHOT-jar-with-dependencies.jar
```
